## HOW TO USE THIS FILE (read before starting work)
================================================================
This file is the single source of truth for pending design work
on the SyncUp project. It is read and updated by Claude in every
design session.

SESSION STARTUP CHECKLIST:
  1. Read DESIGN-BACKLOG.txt (this file) — find the first OPEN item.
  2. Read ANCHOR-DESIGN.txt — load the full design system context.
  3. Work the first OPEN item (or multiple if they are small / related).
  4. Update ANCHOR-DESIGN.txt with all new rules, surfaces, components,
     nav graph edges, haptics, and open-questions changes.
  5. Mark completed items as DONE in this file (change [ ] to [x]).
  6. Bump the anchor version number and update "Last updated" above.
  7. Deliver the updated ANCHOR-DESIGN.txt to the user.

STATUS KEY:
  [x] DONE     — locked, merged into anchor
  [ ] OPEN     — ready to work, pick this next
  [-] DEFERRED — acknowledged, not ready to design yet

ITEM FORMAT:
  [status] Title
           Category | Rule ref (if locked) | Notes
