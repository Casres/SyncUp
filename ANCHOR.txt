## HARD RULES — Round 14 additions (TweaksPanel · Reduced-motion prototype)
================================================================
R14-1. TWEAKSPANEL IS A PROTOTYPE-ONLY COMPONENT. NEVER ship it in
       production builds or reference it outside HTML deliverables.
       Its purpose is to surface demo-state toggles for prototype
       reviewers without cluttering the phone frame itself.

       VISUAL SPEC:
         Closed handle: 32px-tall pill · right:14px · bottom:104px
           (clears simulated tab bar + safe area). bgElevated · radius
           999 · 1px hair border · shadow rgba(0,0,0,.10). Content:
           sliders icon (16px ink3) + "TWEAKS" overline (10/600 ink3).
           Tap → panel opens 150ms easeOut (scale 0.9→1 + opacity 0→1).
         Open panel: fixed · right:14px · bottom:104px · width 220px ·
           radius 14 · bgElevated · shadow 0 12px 40px rgba(0,0,0,.18) ·
           max-height 320px · overflow-y auto.
           Header: "TWEAKS" overline (10/600 ink3) + × icon-btn 32pt hit.
           Body: control rows with 12px vertical padding, 16px horiz.
           Panel closes on × tap or on Esc.

       CONTROL ROW ANATOMY:
         Each row: 40px min-height · label left (12/600 ink2) ·
           control right (32px trailing). Row dividers 1px hair.
         Toggle: 34×20px pill switch. On = accent fill + white knob.
           Off = bgSunken fill + ink3 knob. Snap 120ms easeOut.
         Select pill: tap cycles through values. 10/600 mono accent
           text · accentSoft fill · radius 999 · 6×10 padding.

       STANDARD CONTROLS (every prototype includes these unless noted):
         "Reduced motion" toggle  — when ON, prototype applies
           prefers-reduced-motion fallback rules (A-11, A-21, R13-1):
           200ms easeOut translates, instant backdrops, no spring
           overshoot, no stagger. Does NOT alter the Spinner.
         "Dark mode" toggle       — switches FLOW_TOKENS.light →
           FLOW_TOKENS.dark on the entire phone frame.

       ADDITIONAL CONTROLS: each HTML deliverable declares its own
         extra rows at the top of the control list above the standard
         two (e.g., "Notif count" select, "Offline" toggle, query
         preset picker). Document these per-deliverable in its HTML
         source as a comment block at the top of the Tweaks config.

       BEHAVIOR RULES:
         Changes apply immediately — no "Apply" button.
         State resets to defaults on page reload.
         TweaksPanel MUST NOT interfere with the phone frame's own
           event handlers (pointer-events none on panel backdrop).
         NEVER add Tweaks rows that modify data a reviewer cannot
           see changing (e.g., internal React Query cache TTLs).

## HARD RULES — Round 13 additions (Gesture polish · Offline · Edge cases)
================================================================
R13-1. SHEET DRAG GESTURE IS VELOCITY-FIRST, DISPLACEMENT FALLBACK.
       The NotifSheet drag gesture (on PullDownAffordance or drag
       handle) resolves to a target detent in two phases:

       PHASE A — ENTRY (closed → first detent):
         Triggered by downward drag on PullDownAffordance.
         Sheet follows touch 1:1 from the first pixel of movement.
         Velocity ≥ 0.5px/ms (downward) at release → full directly.
         Displacement < 60px at release → spring back to closed.
         Displacement 60–159px → snap to peek (light haptic).
         Displacement ≥ 160px → snap to full (light haptic).

       PHASE B — INTER-DETENT (peek ↔ full ↔ closed):
         Triggered by drag on the sheet grab handle area (44pt hit).
         Sheet follows touch 1:1; clamps at translateY(12%) (full
         cap) and snaps closed at bottom of screen.
         Velocity-first rule: if |velocity| ≥ 0.7px/ms at release,
           snap-direction determined by velocity sign regardless of
           displacement:
             Downward velocity ≥ 0.7px/ms from full  → peek.
             Downward velocity ≥ 1.2px/ms from full  → close (skip peek).
             Downward velocity ≥ 0.7px/ms from peek  → close.
             Upward velocity   ≥ 0.7px/ms from peek  → full.
             Upward velocity   ≥ 0.7px/ms from full  → no-op
               (already at top; rubber-band 12px then spring back).
         Displacement fallback (|velocity| < 0.7px/ms):
             From peek: down >40px → close; up >40px → full.
             From full: down >80px → peek; down >200px → close.
             Inside thresholds → spring back to current detent.

       SNAP ANIMATION: 320ms spring cubic-bezier(0.34,1.56,0.64,1).
         Backdrop crossfades 280ms easeStd in lockstep with snap.
         Haptic fires at snap moment (not at pointer-up).

       REDUCED-MOTION FALLBACK FOR GESTURE PATH (A-11, A-21):
         Live tracking: unchanged (1:1 follow is not an animation).
         Snap animation: 200ms easeOut translate (no spring, no
           overshoot). Backdrop: instant switch at release.
         NEVER suppress gesture tracking itself — only the snap anim.

       NEVER require a tap shortcut as the only path between detents
         in production — the gesture is the primary affordance.

R13-2. NOTIFSHEET OFFLINE STATE IS SHOWN INLINE, ABOVE THE FEED.
       When the device loses network while NotifSheet is open, or
       when the sheet is opened while offline:
         OfflineBar appears immediately below the sheet header
           (below "Activity" title + action pills, above the first
           NotifGroupHeader). Same OfflineBar component (R5-7).
         A stale-data sub-line appears between OfflineBar and feed:
           "SYNCED {T} AGO" — mono 10/600 ink3. {T} = relative
           time of last successful feed refresh (e.g., "3 MIN AGO").
           Omit if last sync time is unavailable.
         Feed content: show whatever was last loaded (stale but
           visible). NEVER show a full empty state or spinner when
           stale data exists.
         Network-dependent actions are disabled while offline:
           "Accept" / "Decline" on FriendRequestCard,
           "Join" / "Decline" on GroupInviteCard,
           "Plan something" on InboundBroadcastCard.
           Disabled = opacity 0.4 + pointer-events: none.
           NO tooltip required — OfflineBar provides context.
         Local-only actions remain active while offline:
           Swipe-to-reveal (Mute + Dismiss), Mark all read,
           Clear all. These are local state mutations.
         On reconnect: OfflineBar slides out (su-slide-up 240ms
           easeStd). Stale sub-line hides. Feed auto-refreshes;
           newly loaded items animate in via StaggerList.
         A11y: OfflineBar has aria-live="polite".
         NEVER show two OfflineBars simultaneously (E-1).

R13-3. BROADCAST QUEUED OFFLINE TOAST OMITS THE UNDO PILL.
       When a broadcast was written while offline and fires on
       reconnect, BroadcastToast renders with a modified title:
         Title: "Broadcast sent · was queued"  (13/700)
         Sub:   audience label (same as standard toast)
         Leading marker: state dot (Hard Rule 13 unchanged)
         Undo pill: NOT rendered. The broadcast has already fired
           during reconnect; the undo window has passed and
           presenting an undo that cannot succeed is misleading.
         Close X: present. Auto-dismiss: 3200ms. Haptic: success.
         A11y: role="alert" · aria-live="polite" (unchanged).
       NEVER render an Undo pill on a queued-broadcast toast.
       In all other respects the toast is identical to the
       standard BroadcastToast (position, sizing, tokens, border).

R13-4. AUDIENCEPICKERSHEET IN MODE='FRIENDS' SHOWS AN EMPTY STATE
       WHEN THE USER HAS ZERO FRIENDS.
       When mode='friends' and the friend list is empty, the sheet
       body renders an inline empty state instead of the friend list:
         Icon tile: 56×56 · users glyph · bgSunken fill · radius 16.
         Title:     "No friends yet"  (15/700 ink).
         Body:      "Add friends to invite them to events."
                    (13/500 ink2 · centered · max-width 260px).
         No CTA pill — NEVER navigate away from a picker sheet
           to add friends; doing so loses the user's event context.
       Done pill in the sheet header: disabled (opacity 0.4,
         non-interactive) while the selection count is zero.
       Close X remains active so the user can exit cleanly.
       If the user gains friends during the session (optimistic
         add), the empty state replaces itself with the friend list
         without a full remount — state update triggers re-render.
       NEVER show this empty state for mode='types' (friend types
         may legitimately be zero with no action needed from here).

## HARD RULES — Round 12 additions (Notif routing · Quicksets · Mini-profile)
================================================================
R12-1. NOTIFICATION NAVIGATION COLLAPSES THE SHEET FIRST. When
       any notification card tap fires navigation to a destination
       screen, the sheet executes a dismiss transition
       (flow-sheet-down · 240ms · easeStd) concurrent with the
       destination mounting on the activated tab. The sheet does
       NOT remain open behind the destination. Back from the
       destination returns to that tab's top-of-stack at the time
       the sheet closed — not back to the notification sheet.
       The tapped card is marked read at the moment of tap.
R12-2. CUSTOM QUICKSETS ARE SAVED FROM THE AVAILABILITY EDITOR.
       The entry point is a "Save as Quickset" ghost pill in the
       Availability Editor, positioned below the QuicksetGrid
       section. It is enabled only when at least one day is set
       (AvailabilityEntry map non-empty). Tapping it opens
       QuicksetNameSheet. NEVER add a save affordance inside the
       QuicksetGrid itself or on individual tiles.
R12-3. QUICKSETGRID EXPANDS FOR CUSTOM QUICKSETS. Built-in tiles
       always occupy the first 4 positions. Custom tiles follow
       in user-creation order. The grid uses a 2-column layout
       and wraps as needed. No cap on custom count. NEVER hard-
       code the grid at 4 items.
R12-4. BUILT-IN QUICKSETS CANNOT BE RENAMED OR DELETED. Custom
       quicksets may be renamed or deleted via a ⋯ icon-btn
       (18px ink3 · 44pt hit · top-right of tile). Built-in
       tiles have NO ⋯ control. Delete is TwoTapDestructive.
       NEVER render management controls on built-in tiles.
R12-5. PUBLIC MINI-PROFILE IS A SHEET OVER SEARCH. Tapping a
       PEOPLE row body in Search opens QuickProfileSheet stacked
       on top of the Search overlay. The Search overlay remains
       mounted behind it. Closing QuickProfileSheet returns to
       Search. NEVER navigate away from Search to show this sheet.
R12-6. PUBLIC MINI-PROFILE SHOWS LIMITED DATA ONLY. Visible:
       name · handle · photo · bio (if set) · mutual friends
       count + up to 5 avatars · hosted count · attended count.
       NEVER show: availability · friend count · group membership
       · friend type data. These are friends-only fields.

## HARD RULES — Round 11 additions (Co-host)
================================================================
R11-1. CO-HOST PERMISSIONS ARE SCOPED TO TWO ACTIONS ONLY. A co-host
       may: (a) invite additional attendees via "Invite more" →
       AudiencePickerSheet, and (b) edit event details. A co-host may
       NOT remove attendees, manage other co-hosts, or delete the event.
       NEVER render remove controls, ⋯ menus, or type/group filter rows
       to a co-host viewer.
R11-2. CO-HOST IS GRANTED AND REVOKED FROM ATTENDEESSHEET ONLY. The
       host taps ⋯ on any AttendeeRow to open a RowOverflowMenu with:
       "Make co-host" (or "Remove co-host" if already promoted) +
       "Remove from event" (destructive). No other surface grants or
       revokes co-host. The host's own row has NO ⋯ menu — a host
       cannot make themselves co-host or self-remove.
R11-3. CO-HOST BADGE IS VISIBLE TO ALL VIEWERS. Every attendee who
       opens AttendeesSheet sees the CO-HOST chip on promoted rows.
       Visibility of elevated access is intentionally transparent —
       it is NEVER host-only.
R11-4. REMOVE FROM EVENT MOVES INTO ⋯ MENU. In host view, the
       trailing remove icon-btn from R10-7 is replaced by the ⋯
       overflow icon-btn. "Remove from event" inside that menu is
       the tap-to-arm path and fires the same armed → commit flow
       as R10-4. Swipe-to-arm (≥44px rightward) remains a second
       path to arm, unchanged. NEVER keep both a standalone remove
       btn AND a ⋯ menu on the same row.
R11-5. CO-HOST PROMOTION FIRES NOTIFICATION + TOAST. The promoted
       attendee receives: (a) CoHostCard in NotifSheet (persistent,
       survives app restart), and (b) CoHostToast if they are active
       in the app at the moment of promotion. Revocation fires
       CoHostRevokedCard in NotifSheet ONLY — no toast on revoke.
R11-6. CO-HOST ROLE LIVES ON EVENT AS AN ARRAY. Event.coHostIds:
       string[] holds the IDs of all co-hosts for that event.
       isCoHost = Event.coHostIds.includes(currentUser.id).
       Roles are non-overlapping: a viewer is host, co-host, OR
       invitee — never two at once. Host id MUST NOT appear in
       coHostIds. Event Detail edit controls render for
       isHost || isCoHost.
R11-7. ATTENDEESSHEET HAS THREE ROLE VIEWS. Roles and their
       AttendeesSheet capabilities:
         HOST     2-row filter (RSVP + type/group) · search ·
                  "Invite more" · ⋯ menu on all rows (not own).
         CO-HOST  1-row filter (RSVP only) · search ·
                  "Invite more" · NO ⋯ menu · NO type/group row.
         INVITEE  1-row filter (RSVP only) · search ·
                  NO "Invite more" · NO ⋯ menu.
       CO-HOST badge is visible in all three views.

## HARD RULES — Round 10 additions (AttendeesSheet)
================================================================
R10-1. ATTENDEESSHEET IS ROLE-AWARE. The sheet renders one of two
       layouts based on the viewer's role on the event:
         HOST    — filter bar (RSVP + type/group), search, full row
                   with RSVP status, remove action, "Invite more" CTA.
         INVITEE — filter bar (RSVP only), search, row with RSVP
                   status, no remove, no "Invite more."
       Role is determined by Event.hostId === currentUser.id.
       Co-host role is NOT implemented in this round — deferred.
       NEVER show remove or invite controls to non-host viewers.
R10-2. RSVP FILTER CHIPS ARE LOCKED TO 4 VALUES. The RSVP chip row
       contains exactly: All · Going · Maybe · Not going · No response.
       "All" is selected by default and shows the full list.
       Empty sections after filtering use the existing EmptyAttendees
       pattern — NEVER show a zero-count chip as hidden or disabled.
R10-3. HOST FILTER BAR HAS TWO ROWS. Row 1: RSVP chips (same 4+All as
       R10-2). Row 2: Friend Type / Group chips — one chip per type or
       group the host has, plus "All." Rows are independent filters
       that combine as INTERSECTION (show attendees matching BOTH the
       selected RSVP status AND the selected type/group). NEVER merge
       the two rows into one.
R10-4. REMOVE IS TWOTAPDESTRUCTIVE WITH SWIPE-TO-ARM. The armed state
       can be reached two ways: (a) tap the trailing remove icon-btn on
       an AttendeeRow, or (b) swipe the row rightward ≥44px. Both paths
       enter the same armed state — row background shifts to dangerSoft,
       "Remove" label appears, heavy haptic fires. Second tap on the
       armed row confirms removal (success haptic + row animates out
       320ms easeStd). Swipe <44px springs back. Armed state auto-
       dismisses after 4s with no action. NEVER use a modal for remove.
R10-5. "INVITE MORE" OPENS AUDIENCEPICKERSHEET. The host's "Invite
       more" accent pill in the sheet header opens the existing
       AudiencePickerSheet (mode='friends'). Already-invited attendees
       are shown pre-checked and non-interactive. Confirming adds new
       invitees; success haptic fires and rows animate in via StaggerList.
R10-6. SEARCH FILTERS THE VISIBLE LIST ONLY. The search input inside
       AttendeesSheet filters the currently visible list (post-chip-
       filter). It does NOT fire a network search. Matching is
       case-insensitive against name and @handle. No results within
       the filtered set renders EmptyAttendees inline.
R10-7. ROW ANATOMY IS LOCKED. Every AttendeeRow:
         RingAvatar (40px, status ring reflects availability) +
         name (15/600) + @handle (mono 12/500 ink3) +
         RSVPBadge (right-aligned).
       RSVPBadge: pill · 11/600 · radius 999 · 6×10 padding.
         Going      → limeSoft fill · limeInk text
         Maybe      → accentSoft fill · accent text
         Not going  → popSoft fill · popInk text
         No response→ bgSunken fill · ink3 text
       Host view: trailing remove icon-btn (trash, 20px ink3, 44pt hit).
       Invitee view: no trailing action. Tap row → Friend Profile.
       Min-height 64px.

## HARD RULES — Round 9 additions (Onboarding)
================================================================
R9-1.  ONBOARDING IS A SEPARATE NAVIGATION STACK. No tab bar, no
       FlowHeader during any onboarding screen (sign-up, sign-in,
       forgot password). The main app shell mounts ONLY after
       successful authentication. Onboarding uses its own back arrow.
R9-2.  SIGN-UP PROGRESS IS 6 DOTS. Steps 1–6 each show a 6-dot row
       below the back arrow. Active = 8px accent dot. Completed = 8px
       accent at 40% opacity. Upcoming = 8px hairStrong border, no fill.
       6px gap between dots. NEVER show "Step N of 6" copy.
R9-3.  BACK IS ALWAYS PRESENT AND PRESERVES STATE. Every sign-up step
       and forgot-password screen has a 44pt back arrow. Tapping it
       returns to the previous screen WITHOUT clearing form state.
       Exception: Welcome has no back arrow.
R9-4.  PRIMARY CTA DISABLES UNTIL VALID. The primary pill is disabled
       (opacity 0.4, non-interactive) until minimum field validity is
       met. NEVER show an error before the user first attempts submit.
       After a failed submit, inline errors appear and persist.
R9-5.  OTP AUTO-SUBMITS ON 6TH DIGIT. Entering the 6th digit fires the
       submit automatically — no button tap required. The "Verify"
       button remains as a visible fallback only. Show ButtonLoading on
       auto-submit. NEVER require the tap to proceed.
R9-6.  HANDLE IS CHECKED REAL-TIME WITH 400ms DEBOUNCE. After typing
       stops 400ms, fire an availability check. During check: 18px
       spinner trailing. Result: checkmark in limeInk (available) or ×
       in danger + "Handle taken" inline error (taken). Continue CTA is
       blocked during in-flight checks and when result is "taken."
R9-7.  PROFILE PHOTO AND AVAILABILITY ARE ALWAYS SKIPPABLE. Both steps
       MUST have a visible "Skip for now" path. Standard flow: ghost
       pill below primary. Invite-context on availability step: accent
       ghost pill (more prominent placement).
R9-8.  INVITE-FIRST CONTEXT IS STORED BEFORE AUTH BEGINS. When the
       app opens via event invite deep link, InviteContext (eventId +
       eventName) is captured before the auth flow renders. It surfaces
       as an InviteContextBanner on Welcome, adjusts copy/skip
       prominence on Step 6, and routes post-auth to Event Detail
       instead of Home.
R9-9.  SIGN-IN IDENTIFIER FIELD IS FORMAT-AGNOSTIC. The identifier
       input accepts phone number, email address, or @handle — no
       format selector, no tabs. Keyboard type adapts to first
       character: '@' or letter → default; digit → numeric; letter
       string containing '.' → email. NEVER reject a format.
R9-10. FORGOT PASSWORD ENDS AT CONFIRMATION SCREEN. After submitting a
       reset request the user lands on a confirmation screen within the
       forgot-password sub-flow. They exit ONLY by tapping "Back to
       sign in." NEVER auto-navigate away from confirmation.

## HARD RULES — Round 8 additions (Search)
================================================================
R8-1. SEARCH IS A FULL-SCREEN OVERLAY. Entry is the search icon in
      FlowHeader on Home, Friends, and Groups screens. Tap → overlay
      animates up over the current screen with a 280ms spring
      (flow-sheet-up). Input auto-focuses, soft keyboard up. Left
      "Cancel" / back arrow dismisses. NEVER a full route, tab, or
      separate screen — it is a transient overlay over whichever
      screen invoked it.
R8-2. SECTION ORDER IS LOCKED. When a query is present, results
      render in this exact order under mono overlines:
        FRIENDS · PEOPLE · GROUPS · EVENTS
      Empty sections are omitted entirely (no zero-count headers).
      NEVER reorder. NEVER merge. NEVER add a "Top hit" or unified
      results section above this stack.
R8-3. RESULT-ROW ANATOMY IS LOCKED.
      FRIENDS  RingAvatar (with status ring) + name + @handle
               + CategoryBadge → tap row → Friend Profile.
      PEOPLE   RingAvatar (NO status ring — these are not friends yet)
               + name + @handle + "N mutual friends" sub-line + "Add"
               accent pill. Tap pill → request sent, pill swaps to
               "Sent" + success haptic. Tap row body → public mini-
               profile (out of scope for prototype; route stub).
      GROUPS   CoverArt chip (44px) + group name + member count
               + PrivateBadge if private → tap → Group Detail.
      EVENTS   accentSoft 44px calendar tile + event name (2-line
               clamp) + date + RSVP status dot → tap → Event Detail.
      RingAvatar size is 40 in all rows. Row min-height 64.
R8-4. DEFAULT STATE = RECENTS, MAX 7. With no query the overlay shows
      a "RECENT" overline + up to 7 recent terms newest first. Each
      row has a per-row × to remove. A "Clear all" pill sits top-right
      of the section header. Below the list: "Find anyone by @handle"
      hint in ink3. Tapping a recent term populates the input and
      shows CACHED results immediately — no new server call when
      cache is fresh. Editing the term re-triggers a query.
R8-5. EMPTY RESULTS USES EmptySearch. When a query returns zero hits
      across all four sections, render the existing EmptySearch
      pattern unchanged: search icon tile · "Nothing matched" ·
      "Invite by handle" secondary CTA. NEVER add per-section empty
      states; the entire result stack is replaced by EmptySearch.
R8-6. INPUT BAR IS LOCKED. Full-width pill, radius 12, bgSunken fill,
      placeholder "Search friends, groups, events…", magnifying-glass
      leading icon in ink3, clear × inside right edge ONLY when text
      is present (light haptic on tap). NEVER add filter chips,
      scope pickers, or voice-input affordances inside the input.
R8-7. ADD-FRIEND IS THE ONLY ROW-INTERNAL ACTION. PEOPLE row's "Add"
      pill is the sole tap target inside any result row that does NOT
      navigate. All other rows are wholly tappable surfaces with one
      destination each. NEVER add hover/long-press menus, swipe
      actions, or per-row overflow on the Search overlay.

## CO-HOST SURFACE (LOCKED · Round 11)
================================================================
Role resolution (for all event surfaces):
  isHost    = event.hostId === currentUser.id
  isCoHost  = event.coHostIds.includes(currentUser.id)
  isInvitee = !isHost && !isCoHost

Grant flow (host only · R11-2):
  Host taps ⋯ on any AttendeeRow in AttendeesSheet.
  RowOverflowMenu appears (see component spec).
  Items vary by target's current role:
    Not co-host: "Make co-host"  · then: "Remove from event" (danger)
    Is co-host:  "Remove co-host" · then: "Remove from event" (danger)
  Host's own row: no ⋯ menu rendered.
  "Make co-host" tap:
    → target added to event.coHostIds (optimistic)
    → CoHostCard queued to target's NotifSheet
    → CoHostToast fires if target is active in app
    → success haptic · row re-renders with CO-HOST badge
    → menu dismisses.
  "Remove co-host" tap:
    → target removed from event.coHostIds (optimistic)
    → CoHostRevokedCard queued to target's NotifSheet
    → NO toast on revoke (R11-5)
    → light haptic · CO-HOST badge animates off row
    → menu dismisses.
  "Remove from event" tap:
    → menu dismisses → row enters armed state (R10-4 / R11-4)
    → heavy haptic. Second tap commits remove. Success haptic.

AttendeesSheet — header (CO-HOST view):
  Title: "Attendees" 17/800.
  Sub: "{N} invited" 12/500 ink3.
  "Invite more" accent pill trailing (same as host — R11-1a).
  Close X icon-btn · top-right · 44pt.

AttendeesSheet — filter bar (CO-HOST view):
  Row 1 only — RSVP chips: All · Going · Maybe · Not going ·
    No response. NO Row 2 type/group filter (R11-7).

Event Detail edit controls:
  All edit affordances (pencil FAB, "Edit" pill, inline field taps)
  render when isHost || isCoHost. Invitee view is read-only.
  No new component required — condition change only.

CoHostToast (fires on promotion · live app only · R11-5):
  Docked above tab bar (same position as BroadcastToast).
  accentSoft fill · radius 14 · 1.5px accent border.
  Layout: star icon (16px accent) · "You're a co-host" (13/700 ink) ·
    "{EventName}" (11/500 ink2, 1-line ellipsis) · close X.
  flow-fade-up 320ms spring. Auto-dismiss 3200ms.
  A11y: role="alert" · aria-live="polite".
  Success haptic fires WITH the toast (H-5).

Haptics — co-host additions:
  "Make co-host" confirmed     success
  "Remove co-host" confirmed   light
  ⋯ menu open                  light
  ⋯ menu dismiss               light
  CoHostToast fires            success  (same gesture as promotion)

## QUICKSET SAVE/NAME SURFACE (LOCKED · Round 12)
================================================================
Entry point:
  "Save as Quickset" ghost pill · full-width · in Availability
  Editor below QuicksetGrid section.
  Enabled when AvailabilityEntry map is non-empty (≥1 day set).
  Disabled: opacity 0.4 · non-interactive when map is empty.
  Medium haptic on tap (opening a new sheet).

QuicksetNameSheet:
  Bottom sheet · radius 22 22 0 0 · max-height 50%.
  38×4 grab handle centered at top.
  Animation: flow-sheet-up 280ms spring · flow-sheet-down 240ms easeStd.
  Backdrop: 42% black.
  A11y: role="dialog" · aria-modal="true" · focus trap.
        Esc / drag-down >80px → close (light haptic).
  Header: "Name your Quickset" 17/800 · left-aligned.
  Close X icon-btn · top-right · 44pt · light haptic.
  AuthInputField label="Quickset name" · auto-focus · max 24 chars.
  Validation: non-empty AND unique among existing custom quickset
    names (case-insensitive). Name-exists error: "Name already in
    use" · 12/500 danger · triangle-alert leading (A-16).
  Detail hint (below input · non-editable):
    "Applied over the next 30 days" · 12/500 ink3.
  Footer (pinned above safe area):
    "Save" accent PillBtn (disabled until field valid).
    "Cancel" ghost pill · 12px below.

  On "Save":
    Sheet closes (flow-sheet-down 240ms).
    New tile appears in QuicksetGrid via StaggerList entrance
      (base 0ms · 30ms/item · spring · same stagger as list adds).
    Success haptic.
    AvailabilityEntry map unchanged — availability already set by
      the user before entering the sheet. NO "Applied" confirm fires
      on the new tile (nothing new was applied).

Rename flow (custom tiles only · R12-4):

## HARD RULES — v2.7 additions
================================================================
R7-1. QUICKSETS ARE USER-EXTENSIBLE. The 4 built-in presets
      (weekends-free / weekdays-5pm / next30-maybe / clear-month) are
      permanent and may not be deleted. Users CAN save additional
      custom Quicksets beyond the four built-ins. A save/name flow is
      required and will be designed in the next round; until shipped,
      QuicksetGrid renders the 4 built-ins only. NEVER hard-code the
      grid as fixed-length-4.
R7-2. BROADCASTTOAST STAYS SIMPLE. The toast surface contains exactly:
      state dot · title · sub · Undo pill · close X. There is NO
      "review recipients" tap-action on the toast. The toast body
      itself is NOT tappable to open audience editing. Broadcast
      Settings is the SINGLE editor for audience. NEVER add a
      "Tap to review" affordance, a recipient avatar stack, or a
      chevron-to-edit on this component.
R7-3. FRIEND TYPES ARE NON-OVERLAPPING. One friend belongs to exactly
      one FriendType. No nested membership, no compound membership.
      Assigning a friend to a new type REMOVES them from any previous
      type. AudiencePickerSheet (mode='types') does NOT support compound
      filtering. FilterChipRowMulti computes UNION, never INTERSECTION.
      FriendType.members[] is disjoint across types.
R7-4. STALE NOTIFICATIONS AUTO-PURGE AT 30 DAYS. Items older than 30
      days are removed on next sheet open. The EARLIER bucket caps at
      30 days. A manual "Clear all" pill IS in the NotifSheet header
      next to "Mark read." Single-tap (not TwoTapDestructive). Fires
      warning haptic + A11yLive announce ("Activity cleared"). Empties
      feed; EmptyNotifications renders. Per-card left-swipe Dismiss
      handles one-off cases.
R7-5. PUSH COPY = IN-APP COPY VERBATIM. OS-level push strings match
      in-app card title and body exactly. No "New activity from" /
      "SyncUp:" / "You have a new" prefix.
        ✓ "Ana is free tonight"
        ✗ "New activity from Ana"
        ✓ "Sam said yes to Dinner Friday"
        ✓ "Maya wants to be friends"
R7-6. GROUP-DETAIL ADMINBAR IS PINNED. AdminBar stays pinned when
      Group Detail body scrolls. Does NOT collapse, fade, shrink, or
      auto-hide on scroll. Visibility is binary on role only.

## HAPTIC FEEDBACK (LOCKED)
================================================================
6 types only — light / medium / heavy / success / warning / error.
Call once per gesture via useHaptic(). Never on render or scroll.
  light    Tab change · mode switch · brush change · filter chip · day cell tap
           · open Search overlay · tap Search result row · clear input ×
           · tap recent search term
           · QuickProfileSheet open/close · QuicksetNameSheet close X
           · Quickset ⋯ menu open/dismiss · delete arm auto-dismiss
  medium   RSVP confirmed · step advance · send invites · Quickset applied
           · toggle flipped · Done on picker
           · notification card tap → navigate (R12-1; all nav cards except
             InboundBroadcastCard body which stays light)
           · "Save as Quickset" pill tap
  heavy    TwoTapDestructive armed · long-press chip reaches 450ms
           · drag-paint hits same-brush day (no-op)
           · Quickset delete armed
  success  Event created · friend accepted · invites sent · broadcast fired
           · cover saved · group created · Quickset Applied confirm
           · Search Add → Sent (PEOPLE row friend request)
           · Quickset saved (new) · Quickset renamed · Quickset deleted
           · QuickProfileSheet "Add friend" confirmed
           · QuickProfileSheet "Accept" friend request
           · BroadcastToast queued=true (broadcast sent on reconnect)
  warning  Step 3 busy banner · RSVP Maybe on busy day
           · Quickset overwrites >5 days · offline entered
           · NotifSheet Clear all · Search Clear all recents
  error    RSVP failed · invite failed · friend request failed
           · broadcast failed · network error
Hard rules:
  H-1. NEVER haptic on scroll, drag-momentum, or visual transitions.
  H-2. NEVER stack haptics within 80ms (debounce in useHaptic).
  H-3. NEVER haptic on mount or auto-fired toasts not triggered by user.
  H-4. Two-tap destructive: heavy on arm, success on commit.
  H-5. Toasts fire haptic WITH the toast, not on dismiss.

## ACCESSIBILITY RULES (LOCKED)

  A-1.  ink3 = AA Large only. Use ink2 below 18px (or 14px bold).
  A-2.  Availability state = AvailDot + text label always (R5-1).

  A-5.  Minimum 44×44pt. Filter chips: visual 30px, hit 44px via invisible padding.

  A-7.  All icon-only buttons carry aria-label.
  A-8.  Spinner: role="status" + aria-label="Loading".
  A-9.  Toasts: role="alert" + aria-live="assertive" (errors) or "polite" (broadcasts).
  A-10. Use A11yLive for ephemeral status with no visible toast.
Motion:
  A-11. prefers-reduced-motion: no spring overshoot, opacity fade 200ms easeOut,
        no stagger (all-at-once), keep spinner.
  A-12. No parallax or scroll-bound transforms (R5-3).
Text:
  A-13. Long-text rules R5-6 apply system-wide.
  A-14. Text-only tap targets ≥14/600 for 44pt with vertical padding.
Forms:
  A-15. Every input has associated label or aria-labelledby.
  A-16. Input errors use popInk + triangle icon, never color alone.

  A-17. Full detent: role="dialog" + aria-modal="true".
        Peek: role="region" + aria-label="Activity preview".
  A-18. Focus trap on full detent only. Esc collapses one detent.
  A-19. SR order = visual order. NotifGroupHeader aria-level="3".
  A-20. Dismiss = triple-redundant: drag handle + Esc + close X.
  A-21. Reduced-motion: 200ms easeOut translate (no spring), instant
        stack expand, tap-only affordance, instant backdrop.
  A-22. PullDownAffordance announces count change via aria-live="polite".
Search overlay (Round 8):
  A-23. SearchOverlay uses role="dialog" + aria-modal="true".
        Input auto-focuses on open; focus returns to invoking icon on close.
  A-24. Cancel button is always reachable as first tab stop.
  A-25. RecentRow × buttons carry aria-label="Remove {term}".
  A-26. "Add" pill carries aria-label="Send friend request to {name}";
        "Sent" state announces via aria-live="polite".
