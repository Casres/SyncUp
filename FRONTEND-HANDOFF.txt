## STATE MANAGEMENT RULES (hard — do not deviate)
================================================================
DATA RULE 1: If data comes from the API, it lives in React Query.
             No exceptions. Never copy API responses into Zustand
             or any other store.

DATA RULE 2: If data never touches the network, it lives in
             local component state (useState / useReducer) or
             draftStore for the create-event flow.
             No Zustand or global store for UI-only state.

DATA RULE 3: Never store API response data manually in Zustand.
             (There is no Zustand in this project — if you find
             yourself reaching for it, you're off-pattern.)

See src/api/ for all query keys, query functions, and the
configured queryClient. Mock responses live in src/mocks/.

## TAB BAR (LOCKED — R6-6, Hard Rule 23)
================================================================
Tab order is fixed and must never change:
  1. Home        (house icon)
  2. Broadcast   (megaphone / antenna icon)
  3. Activity    (bell icon)  ← opens NotifSheet at full detent
  4. Friends     (people icon)
  5. Profile     (person icon)

Unread dot on Activity tab = silent icon dot only (no copy).
Home pull-to-reveal pill = "N NEW" verbal affordance.
Both share unread count state and clear in lockstep (R6-5).

GAP 7: NOTIF SHEET GESTURE UPGRADE  (spec: Round 13 · R13-1)
────────────────────────────────────────────────────────────────
The existing NotifSheet uses useSheetDrag. It needs to be updated
to the velocity-first snap model:

PHASE A (closed → first detent via PullDownAffordance):
  velocity ≥ 0.5px/ms downward → snap full (light haptic)
  displacement <60px → spring back to closed
  displacement 60–159px → snap to peek (light haptic)
  displacement ≥160px → snap to full (light haptic)

PHASE B (inter-detent via grab handle):
  velocity-first at ≥0.7px/ms overrides displacement.
  Velocity table:
    downward ≥0.7 from full   → peek
    downward ≥1.2 from full   → close (skip peek)
    downward ≥0.7 from peek   → close
    upward   ≥0.7 from peek   → full
    upward   ≥0.7 from full   → rubber-band 12px, spring back
  Displacement fallback (<0.7):
    peek → down>40px → close; up>40px → full
    full → down>80px → peek; down>200px → close
    inside thresholds → spring back to current detent

Snap animation: 320ms spring cubic-bezier(0.34,1.56,0.64,1)
Backdrop: crossfades 280ms easeStd in lockstep.
Haptic fires at snap moment (not at pointer-up).

Reduced-motion fallback (A-11, A-21):
  Live tracking: unchanged (1:1 follow, never suppressed).
  Snap animation: 200ms easeOut (no spring, no overshoot).
  Backdrop: instant at release.

## KEY RULES QUICK REFERENCE (read ANCHOR for full spec)
================================================================
Haptics (R5-8): Only 6 types — useHaptic('light'|'medium'|'heavy'|
  'success'|'warning'|'error'). No raw Haptics.* calls outside that
  hook. Never haptic on InboundBroadcast receive (R6-3).

Loading states (R5-2): Spinner ONLY. No skeletons, no shimmer.
  Spinner = rotating arrows, 900ms linear. role="status".

Destructive actions (Hard Rule 7): ALL destructive flows use
  TwoTapDestructive. No confirmation modals. Exception: "Log out"
  uses OS confirm in production.

Truncation (R5-6):
  Event title: 2-line clamp
  Friend name: 1-line ellipsis
  Description: 3-line clamp + "Read more"
  @handle: 1-line ellipsis (mono)

Availability state: never store null for a day. Clearing a day
  DELETES the key from the AvailabilityEntry map (Hard Rule 14).

Friend Types are non-overlapping (R7-3). Assigning a friend to
  a type removes them from any prior type. Never compound membership.

Push notification copy = in-app card copy verbatim (R7-5).
  Never prefix "New activity from" or "SyncUp:".

AdminBar on Group Detail is ALWAYS pinned (R7-6). Never collapses.

Notifications purge at 30 days on sheet open (R7-4).
  "Clear all" is single-tap (not TwoTapDestructive). Warning haptic.

Search is a full-screen overlay, not a screen or tab (R8-1).

NotifSheet detents: exactly peek (44%) and full (88%) (R6-2, Hard
  Rule 19). Pull from Home → peek. Activity tab tap → full.

TweaksPanel (R14-1): prototype HTML only. NEVER ship in production.
  Do not reference this component outside HTML deliverables.
