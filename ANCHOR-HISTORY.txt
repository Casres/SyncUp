(Edits & tweaks made during the polish pass. Logged here for
traceability; folded back into component definitions in ANCHOR.txt.)
  SettingsRow
    · Renders <div> when no onClick (was always <button>). Fix for
      Toggle-in-button-in-button warning. Status: Hard Rule 16.
  FilterChipRow / FilterChipRowMulti
    · Visual height stays 30px; tap target raised to 44px via invisible
      vertical padding (touch-action: manipulation).
    · Long-press hint: 450ms · linear · subtle scale to 0.98.
  RingAvatar
    · Now ALWAYS rendered alongside an AvailDot or text label when
      conveying state (R5-1). Ring color/arc alone is decorative.
  BroadcastToast / ErrorToast
    · Both gained role="alert" + aria-live (R5-4 / A-9).
    · Error variant: leading marker is a popSoft alert-triangle, not a
      state dot. Broadcast variant unchanged (state dot).
  Spinner (NEW · R5)
    · The ONLY loading affordance. Replaces all skeletons.
    · Always rendered from first mount.
  OfflineBar (NEW · R5)
    · Color + icon + text triple. Slides down from FlowHeader.
    · Warning haptic on entrance, no haptic on auto-hide.
  StaggerList (NEW · R5)
    · 30ms per item · base 60ms · spring · cap 12 items.
    · Used on Friends list, Groups list, Notifications feed, Empty
      state inventories. NOT used on day grids (would feel laggy).
  ThemePicker (R4)
    · Confirmed accessible: each cell has aria-pressed; active cell
      passes 4.5:1 contrast on bgElevated.
  Step3InviteWired (R4 wire-back)
    · Danger banner gained role="alert" — announce-on-mount.
    · Free/Maybe overline got AvailDot so it's not color-only (R5-1).
  PillBtn / Toggle / FormField
    · No changes; all met R5 audit on first pass.

## CHANGES MADE IN ROUND 4
================================================================
  – Initial filename was double-encoded ("Profile &amp;…html"); renamed via
    move to "Profile & Availability.html".
  – SettingsRow originally rendered as <button> always, which produced a
    React validateDOMNesting warning when the trailing slot held a Toggle
    (button-in-button). Refactored to render <div> when no onClick is
    provided. This is now Hard Rule 16.
  – Broadcast IA approved before build (3-card stacked pattern); no
    deviations during implementation.
  – Wire-back was added as a new Step 3 variant (Step3InviteWired) rather
    than mutating the original Step3Invite — original remains untouched
    and still renders the locked Banded viz.

## CHANGES MADE IN ROUND 5 (Polish Pass)
================================================================
  – Skeletons were initially prototyped (shimmer + pulse variants) but
    cut after motion review: a real-time app with variable-shape data
    creates layout-whiplash when fakes don't match real. Replaced with
    spinner-only (R5-2). Skeleton tokens (skeletonBg, etc.) deleted from
    FLOW_TOKENS to prevent regression.
  – Initial haptic registry had 9 types (added selection / impact-soft /
    impact-rigid). Trimmed to 6 to match expo-haptics 1:1; the extras
    were aliases that drifted between iOS/Android. Locked at 6.
  – First pass on ErrorToast used a state-colored dot (matched broadcast).
    Felt indistinguishable from a successful broadcast at a glance.
    Switched to popSoft alert-triangle so the variant is unmistakable.
  – Filter chip carousel originally failed 44pt (visual 30px == hit 30px).
    Fix: invisible vertical padding 7px each side bumps hit area to 44pt
    while keeping visual height. Documented as A-5 inventory.
  – ink3 contrast audit caught 4 surfaces using ink3 below 14px (toast
    subtitle, broadcast card body, settings row sub, filter chip count).
    All swapped to ink2. Codified as R5-5.
  – Stagger entrance was originally on day grids; pulled — felt laggy on
    a 30-cell month view. Now restricted to lists ≤ 12 items.
  – Anchor doc bumped to v2.5 (this update). No code changes — doc-only
    refresh promoting Round-5 details to canonical reference.

## CHANGES MADE IN ROUND 6 (Notifications & Activity)
================================================================
  – First sketch had Activity as a full-screen tab feed AND a separate
    pull-down peek-only sheet on Home. Cut after IA review: same data
    surfacing through two divergent layouts created split-brain affordances.
    Consolidated to ONE sheet · TWO entry points (R6-1, Hard Rule 18).
  – Initially explored 3 detents (peek / mid / full). Cut — the mid
    detent landed in a deadzone where neither the calendar nor the feed
    was usable. Locked at 2 (R6-2, Hard Rule 19).
  – Pull-down affordance prototyped as an invisible drag-zone (no visible
    bar). Tested poorly — users didn't discover it. Replaced with a
    visible PullDownAffordance pill that carries the unread copy too,
    solving discovery + unread state in one element.
  – Activity tab unread dot first prototyped as a numeric badge ("3").
    Conflicted with Home pull pill ("3 NEW") — felt double-counted.
    Switched to a count-less pop dot; Home pill is the only place the
    number appears. Locked as R6-5 / Hard Rule 22.
  – Inbound broadcast cards initially fired a light haptic on arrival,
    matching outbound BroadcastToast. Cut after testing: a chatty friend
    triggered haptic spam. Promoted to hard rule R6-3 / 20 — never
    haptic on receive, only on user interaction. Aligns with H-3.
  – Stack-by-sender window started at 15 minutes. Too tight — typical
    "I'm free in 30min" scenarios produced 2–3 separate cards from the
    same person. Widened to 60 min. Locked as Hard Rule 21.
  – "Mark all read" was a destructive TwoTapDestructive at first.
    Overkill — marking read is not destructive (items remain visible).
    Downgraded to a single-tap pill in the sheet header.
  – "Clear all" button was prototyped, then cut. Reversed in v2.7 —
    Clear all IS now in the sheet header (R7-4).
  – FGTabBar (4-tab) is retained for the Round-3 prototype shell only.
    All new screens use N6TabBar. Documented in Tab Bar IA → Deprecated.
  – Anchor doc bumped to v2.6 (this update).
